1. Property browser — Point / PointF managers

Goal. Add compound property managers that map cleanly to QPoint, QPointF, cv::Point, cv::Point2f, cv::Point2d, cv::Point3i, cv::Point3f, cv::Point3d, with x/y (or x/y/z) sub-items.

Pattern adopted. Mirror the existing PositionPropertyManager / SizePropertyManager style — two manager classes, mode-switched.

User-facing nameManager + mode
PointXY PointPropertyManager + XY
PointXYZ PointPropertyManager + XYZ
PointXYDouble PointFPropertyManager + XY
PointXYZDouble PointFPropertyManager + XYZ

Files.

  • src/widgets/property_browser/custom_property_managers.h — added two manager classes with setValue overloads for the OpenCV / Qt point types, gated by #if __has_include(<opencv2/core/types.hpp>) so the header stays portable.
  • src/widgets/property_browser/custom_property_managers.cpp — implementations mirroring PositionPropertyManager (init/uninit, sub-prop creation/destruction, slotIntChanged / slotDoubleChanged, valueText).
  • src/widgets/property_browser/property_browser_widget.{h,cpp} — added accessors pointManager() / pointFManager(); registered QtSpinBoxFactory + QtDoubleSpinBoxFactory for the sub-managers.

4. Patterns widget — property-rebuild bug

Bug. Clicking a pattern in the tree silently failed to render the property widget (or crashed on some builds).

Root cause. MatchConfigPropertyAdapter::destroy() called m_mgr->clear(). The variant manager is shared with LocalizationPatternsWidget's group-level properties; clear() deletes every property in the manager. Inside rebuildPropertyBrowser() the second adapter bind() wiped the freshly-created group properties → adding a dangling m_groupVariant to the browser.

Fix.

  • src/matching/match_config_property_adapter.cpp::destroy() — now only deletes properties the adapter itself created (qDeleteAll(m_props) + delete m_grpCommon + delete m_grpType). Shared manager untouched.
  • src/form/task/localization_patterns_widget.cpp::rebuildPropertyBrowser() — owns the lifecycle of m_groupVariant: explicit qDeleteAll(m_groupProps); delete m_groupVariant; before re-creating.

5. onPropertyValueChanged — commit to manager

Before. The method only had a half-finished branch; the condition was m_groupPropKeys.contains(property) || (m_boundMatchGroup) (should be &&, saved only by key.isEmpty() return;). No null guards, no failure rollback, no UI re-sync.

After.

  1. Guard: only handle group props; require m_patternManager + m_boundMatchGroup.
  2. PropSpecHelper::dispatch(...) into the working copy.
  3. Snapshot identity (oldName) before the commit — dispatch may have already written a new name into the working copy.
  4. setGroupConfig(oldName, m_workingGroupConfig):
    • Failure → roll the working copy back from the live group, call PropSpecHelper::refresh() so the browser stops showing the rejected value.
    • Success → let the manager's groupChanged listener handle UI refresh (after the cleanup in §6).

6. Cleanup — manager owns signal emission

Issue. MatchGroup::setConfig had a commented-out emit configChanged, but MatchGroup is not a QObject so the emit could never work. As a result setGroupConfig / setPatternConfig / etc. all silently succeeded without notifying listeners.

Manager now emits the right signal on every successful mutation (src/matching/pattern_group_manager.cpp):

MethodSignal
setGroupConfig groupChanged(group, "*")
renameGroup groupChanged(group, "name")
renumberGroup groupChanged(group, "number")
setPatternConfig patternChanged(group, p, "*") (re-resolves p by new name)
renamePattern patternChanged(group, p, "name")
renumberPattern patternChanged(group, p, "number")
setPatternImage patternChanged(group, p, "image")

Listeners updated in src/form/task/localization_patterns_widget.cpp:

  • groupChanged listener syncs m_selectedGroupIndex (when bound), calls rebuildTreeFromManager() + rebuildGroupCombo(), and re-selects by the possibly-updated number.
  • patternChanged listener refreshes the tree + thumbnail for the bound pattern.
  • onPropertyValueChanged collapsed: no longer manually rebuilds tree/combo — the signal listeners do it.
  • onPatternConfigModified got the same rollback path: on failure, restore m_workingPatternCfg from m_boundPattern->config() and call m_configAdapter->refresh().

7. JSON save/load for TaskLocalization

Goal. Implement toJson/fromJson for the task + serialise the entire pattern library, with training images stored as BLOBs through ProjectRepository.

Critical blocker fixed in task_factory.cpp. Even after correct fromJson, TaskFactory::createTaskLocalization only called config.fromJson(...); it never invoked task->fromJson(obj). Result: cameraDeviceId, mcDeviceId, assignedDeviceIds, and the entire pattern library were silently dropped on load. Replaced the partial restore with a single delegated call: 
TaskLocalization* task = new TaskLocalization(taskName, taskId, parent);
if (!task->fromJson(obj)) { /* log + keep task anyway */ }

PatternGroupManager::toJson / fromJson

Goal. Move pattern-library serialisation into the manager so it owns its own schema; have TaskLocalization delegate.

Files.

9. Image BLOB I/O for the task

TaskLocalization::getTaskImageMap / loadTaskImageMap in src/model/task_localization.cpp:

  • getTaskImageMap: walks every group → pattern, includes only patterns whose m_rawImage is non-empty, keyed by imageKey(gn, pn).
  • loadTaskImageMap: parses each key with a QRegularExpression, finds group/pattern by number, writes the image through setPatternImage(...). Bad keys / missing groups / missing patterns are logged via LOG_DEV_ERR; the rest still inject.

Stable key "g{N}_p{M}" — uses numbers, not names, because names can be renamed at runtime without a corresponding storage update.

10. AddPatternWizard canvas refactor

Goal. Add canvas-wide pan/zoom/reset, crop body drag, read-only crop overlay on the Pick step, and box jaw handles.

Foundational change. Previously all geometry was in widget pixels. Refactored so every public state member is in source-image pixels and the canvas maintains a view transform (m_zoom, m_offset). Paint applies image → widget at render time; interactions convert mouse → image via widgetToImage().

View interaction (every mode)

GestureEffect
Mouse wheel Zoom around cursor (point under cursor stays fixed). Clamped [0.05, 40]×.
Middle-button drag Pan (works even when locked).
Middle-button double-click resetView() — fit to widget + recentre.

setImage() auto-fits when a new image arrives. resizeEvent is a no-op so the user's view survives widget resize.

Crop mode

hitCropHandle() returns one of CH_TL/TR/BL/BR/Body. Press → start drag, move → apply image-space delta:

  • Corners → setTopLeft/TopRight/...
  • Body → translate(delta)

Cursor reflects handle (SizeFDiagCursor, SizeBDiagCursor, SizeAllCursor). Min size 8×8 image pixels; intersected with image bounds. Emits cropChanged(m_crop).

Pick mode

  • Paints m_crop as a dashed read-only outline with dim scrim; not interactive.
  • Click clamps the pick into the crop rect (or image bounds if no crop).
  • Wizard passes m_keepOriginal ? QRect() : m_crop to hide the overlay when keepOriginal is on.
  • AddPatternWizard::pickX/Y() accessors return crop-relative when cropped, raw image coords otherwise.

Box mode

  • 4 corner handles → resize. Cursor → box-A-local axis-aligned coords; new W = 2·|lx|, H = 2·|ly|. Box A stays anchored; box B mirrors.
  • Body drag → translate the jaw pair. Polar coords from the pick set m_boxDist + m_boxAngle.
  • Rotation handle (filled disc on the perpendicular line from box A centre, m_boxH/2 + 28 px standoff) → drag rotates around the pick.
  • Boxes drawn without clipping to the image rect → jaws can extend off the frame.
  • New signal boxChanged(w, h, dist, angleDeg) emitted on every change; wizard listens, updates spin boxes (signal-blocked) and state.

11. Wizard polish — image-coord contract + spin-box sync

  1. QPainter::SmoothPixmapTransform is OFF. Antialiasing stays on for overlays; the underlying pixmap renders nearest-neighbour so pixel boundaries are visible at high zoom.
  2. All canvas signal payloads are in image pixels — documented inline in pattern_canvas.h. Verified every emission (cropChanged, pickChanged, boxChanged) uses image-coord members.
  3. Bidirectional sync canvas ↔ widget objects. The wizard already synced spin boxes from canvas signals, but they were bounded by legacy CW=560 / CH=380 constants — values outside silently clipped. New helper AddPatternWizard::onImageSizeChanged(imageW, imageH) resizes every geometry spin box on image load:
    • m_cropX/Y → [0, imgW/H]; m_cropW/H → [1, imgW/H]
    • m_pickXSpin/YSpin → [0, imgW/H - 1]
    • m_boxWSpin/HSpin → [1, max(maxDim, 5000)]
    • m_boxDistSpin → [0, max(hypot(W, H), 1000)]
    • All setRange / setValue inside per-spin QSignalBlocker so no valueChanged → onCropChanged cascade fires.
    • Re-centres / clamps m_crop and m_pick when previous defaults fall outside the new image bounds.

onResetCrop / onCenter1to1Crop / onPickCenter now use image dimensions (m_capturedMat.cols/rows) rather than canvas constants.

Known follow-ups / open items

  • ITask::fromJson reads assignedDeviceIds but does not read cameraSourceType / ownedCameraId, even though ITask::toJson writes them. Pre-existing base-class gap.
  • LocalizationCalibrationWidget shares the manager with the patterns widget but does not currently listen to its signals; cross-widget refresh on simultaneous edits is missing.
  • Patterns come back from a load with m_hasPatternLearned == false — re-learn happens lazily at first use (intentional).
